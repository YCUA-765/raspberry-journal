## 🔴 踩坑实录

### 第一坑：命令名字变了

```bash
libcamera-hello
# → command not found
```

树莓派5 + 新版系统已经把命令从 `libcamera-*` 改成了 `rpicam-*`，正确命令是：

```bash
rpicam-hello
```

老教程和文档很多还在用旧命令名，遇到 `command not found` 先确认命令名有没有变。

---

### 第二坑：apt update 在校园网下失败

```
The repository is no longer signed. Failed to fetch, got 'Nosplit'
```

校园网会拦截和修改网络请求，导致软件源的数字签名验证失败——系统下载回来的软件包列表已经被动过手脚，和官方签名对不上，所以拒绝。

切换到手机热点（关闭 VPN）之后恢复正常。

---

### 第三坑：config.txt 修改没保存进去

在 nano 里编辑 `/boot/firmware/config.txt` 添加 `dtoverlay=imx219` 之后，以为保存了，但实际上文件没有被修改。

用这个命令确认有没有真的保存：

```bash
cat /boot/firmware/config.txt | grep -i camera
```

如果搜不到就说明没保存成功，需要重新来一遍。nano 保存时注意底部显示的文件名是否正确，不要误存到了别的文件名。

---

### 第四坑：CAM0 检测不到

插在 CAM0 接口上，`rpicam-hello` 一直报 `no camera available`。

---

## ⌨️ 小屏幕 + 虚拟键盘的使用技巧

没有购买独立键盘，只能用触摸屏虚拟键盘输入。7寸小屏幕上虚拟键盘弹出来之后会遮住终端下半部分，导致命令输出看不到。

解决方法：**把终端窗口设为全屏**。全屏状态下终端会自动适应键盘的高度，始终把最后一行输出保持在键盘上方可见区域，不会被遮住。

## 🔍 排查过程：三步定位问题

### Step 1：确认不是树莓派的问题

```bash
ls /dev/video*
# → /dev/video18  /dev/video19  /dev/video20 ... /dev/video35
```

系统检测到了大量 video 设备，说明树莓派本身没有问题，摄像头在硬件层面是被感知到的。

> 💡 Linux 里一个摄像头会产生多个 `/dev/video` 节点，每个节点对应摄像头的不同功能模块（图像数据、参数控制、ISP处理等），所以看到十几个是正常的。

### Step 2：通过 dmesg 报错定位问题层级

```bash
dmesg | grep -i imx219
# → Error reading reg 0x0000: -121
# → error -EREMOTEIO: failed to read chip id
# → imx219 probe with driver imx219 failed with error -121
```

这个报错说明：内核找到了 IMX219，尝试和它通信，但通信失败了（`-EREMOTEIO` = Remote I/O Error，远端设备没有响应）。

问题范围缩小到：**树莓派和摄像头之间的物理连接**，不是软件问题。

### Step 3：红灯排除摄像头本身损坏

调试过程中注意到摄像头芯片会亮红灯——之前以为红灯是故障信号，其实不是。红灯表示**摄像头有通电**，说明摄像头本身没有损坏，只是初始化失败后熄灭。

问题进一步缩小到：**CAM/DISP 接口的排线连接质量**。

---

## 📌 CAM 口和 DISP 口的区别

树莓派5上有两组外观相似的排线接口，最开始没有分清：

| 接口 | 用途 |
|------|------|
| CAM0 / CAM1 | 专门连接摄像头（Camera） |
| DISP0 / DISP1 | 连接外接显示设备（Display） |

两者外形几乎一样，但用途完全不同，摄像头排线必须接 CAM 口，接到 DISP 口是不会被识别的。

---

## 🔧 关键发现：锁扣

排查硬件连接时发现 CAM/DISP 接口有一个很小的**锁扣机制**：

```
插排线前：锁扣向上抬起（松开状态）
插入排线：金属触点朝向凹槽，插到底
插排线后：锁扣向下按紧（锁定状态）
验证：轻拉排线拉不动 = 锁好了
```

之前一直以为直接插进去就行，没有注意到锁扣，导致排线实际上没有固定好，接触不稳定。发现锁扣之后重新插好，极大提高了连接成功率。

---

## ✅ 解决方案

**换到 CAM1 接口 + 正确锁紧排线**，同时在 config.txt 里指定：

```
dtoverlay=imx219,cam1
```

重启后 `rpicam-hello` 成功弹出预览窗口。

画面是倒置的（摄像头安装方向问题），加参数旋转：

```bash
rpicam-hello -t 0 --rotation 180
```

画面方向正常 ✅

## 💭 今日总结

**技术层面**

- **命令名变更**：树莓派新系统 `libcamera-*` → `rpicam-*`，遇到 `command not found` 先查命令名
- **dmesg 的价值**：系统日志是定位硬件问题的关键，`-EREMOTEIO` 直接指向通信层故障
- **排查逻辑**：从大到小缩小范围——先排除树莓派（`/dev/video*`），再排除摄像头本身（红灯），最后定位到接口连接

**硬件层面**

- CSI 排线接口有锁扣，不锁紧就是虚接，这是这次调试最关键的发现
- 红灯 ≠ 故障，是通电信号

**认知层面**

第一次完整走了一遍"硬件不工作"的排查流程。和软件报错不同，硬件问题没有明确的错误信息直接告诉你哪里坏了，需要一层一层排除：系统层 → 驱动层 → 物理连接层。`dmesg` 是这个过程里最有用的工具。

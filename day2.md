## 💣 起因：定制镜像的"黑盒"

昨天尝试在厂家预烧录的定制镜像里添加 `ssh` 文件，结果发现文件总是被系统自动删除。考虑到定制系统可能存在未知的权限限制或特殊机制，决定放弃"修补"路线，改用官方镜像从头开始，以确保环境的纯净和可控。

---

## 🔴 踩坑实录：三次烧录记

### 第一次：大意失荆州

**操作：** 下载 Raspberry Pi OS (Bookworm)，使用 Imager 烧录。为了求快，一路点击 "Next"，跳过了"编辑设置"。

```bash
ssh pi@raspberrypi.local
# → Connection refused
```

**原因：** 官方镜像默认不开启 SSH。

---

### 第二次：老教程的"陷阱"

**操作：** 重新烧录后，按照网上"经典老教程"，在 `bootfs` 分区手动创建空的 `ssh` 文件。

**结果：** 文件创建成功，但连接依然失败。

**反思：** Bookworm 系统对此有了新的处理机制，且没有配置网络，树莓派根本没连上网。

---

### 第三次：正解出现 ✅

**关键发现：**

> Raspberry Pi OS (Bookworm) 已经**不支持**在 SD 卡里直接放 WiFi 配置文件了！

这是一个重大的版本变更。旧系统支持在 boot 分区放 `wpa_supplicant.conf` 来配置 WiFi，但新系统必须在烧录阶段就把 WiFi 信息写进去。

**正确操作：**

1. 打开 Raspberry Pi Imager
2. 选好系统和存储设备
3. 点击 **"Edit Settings"**（不要跳过！）
4. **General 标签页**：设置用户名、密码、WiFi（填手机热点的 SSID 和密码）
5. **Services 标签页**：勾选 **Enable SSH**
6. 保存设置，开始烧录

**结果：**

```bash
ssh pi@raspberrypi5.local
# pi@raspberrypi5:~$  ← 连接成功！🎉
```

## 🤖 惨痛教训：AI 的"盲人摸象"

在第三次烧录成功之前，我经历了一段最糟糕的"盲目寻址"过程。

### 发生了什么

为了不重烧系统，我试图在复杂的网络环境中找到树莓派的 IP，询问 Claude 如何排查。它给出了一个"猜测式"的方案：

- 推测树莓派可能在某个网段（如 `192.168.x.x` 或 `10.42.0.x`）
- 让我逐个尝试 SSH 连接来判断 IP

```bash
ssh pi@192.168.1.1
ssh pi@192.168.1.2
# ...
```

### 为什么这是灾难

| 问题 | 说明 |
|------|------|
| 效率极低 | 网段内有 200+ 个可能地址，逐个尝试是大海捞针 |
| 逻辑缺陷 | 完全忽略了网络拓扑的复杂性（校园网、热点、多网卡） |
| 浪费时间 | 像个无头苍蝇一样试了半天，毫无进展 |

### 深刻反思

> **大模型没有"真相"，它只是根据概率生成"看起来像答案"的文字。**

如果你懂网络原理，你会立刻说："应该用 `nmap` 或 `arp` 扫描，或者直接看路由表。"

如果你不懂，你就会盲目执行它给出的低效方案，被它"欺负"。

**结论：自己掌握基础知识，具备判别对错的能力，是使用 AI 的前提。AI 可以是优秀的参谋，但绝不能把决策权完全交给它。**

## 💡 核心收获：Bookworm 的范式转移

| | 旧方法（Bullseye 及以前） | 新方法（Bookworm）|
|---|---|---|
| 开启 SSH | `touch /boot/ssh` | Imager → Edit Settings → Enable SSH |
| 配置 WiFi | `nano /boot/wpa_supplicant.conf` | Imager → Edit Settings → Configure wireless LAN |

> ⚠️ 很多网上教程还在用旧方法，遇到问题先确认系统版本。

## 💭 今日总结

**技术层面**

- **Bookworm 变更**：`wpa_supplicant.conf` 方法已失效，必须用 Imager 预配置网络
- **寻址策略**：网络环境复杂时，不要在乱麻中"寻找设备"。让设备**主动连接**到已知网络，远比扫描更高效

**认知层面**

- **版本意识**：遇到问题先确认系统版本，很多"标准教程"可能已经过时
- **人机协作**：AI 是副驾驶，你才是机长。在缺乏领域知识时，AI 可能会把你带进沟里。保持批判性思维，先建立自己的知识框架，再让 AI 辅助填空
